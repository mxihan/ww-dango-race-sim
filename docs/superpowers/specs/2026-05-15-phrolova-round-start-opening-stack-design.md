# 弗洛洛回合开始判定与默认起点堆叠设计

## 背景

当前没有传入 `starting_state` 时，普通团子不会在比赛初始化时进入棋盘，而是在各自第一次行动前通过 `enter_at_start()` 进入 0 号地块。这个行为会导致第一回合开始时不存在普通团子堆叠，因此依赖堆叠状态的回合开始技能无法在开局触发。

当前弗洛洛技能在 `before_move` 阶段判定：当弗洛洛行动前位于当前堆叠底部，且堆叠至少有两个团子时，本次移动 +3。新规则要求该判定提前到回合开始，也就是所有团子开始本回合行动之前。

## 目标

- 没有传入 `starting_state` 时，第一回合先根据行动顺序骰确定行动顺序，再把所有普通团子堆叠到 0 号地块。
- 这个默认起点堆叠必须发生在第一回合的回合开始技能判定之前，因此可触发弗洛洛、奥古斯塔等依赖回合开始堆叠状态的技能。
- 起点堆叠顺序为：第一回合行动顺序中最先行动的团子在最上面。由于 `RaceState` 使用 bottom-to-top 列表，若行动顺序为 `A -> B -> C`，0 号地块堆叠应保存为 `["C", "B", "A"]`。
- 传入 `starting_state` 时保持现有行为，不额外创建默认起点堆叠。
- 布大王仍由引擎管理，并继续遵守前两回合不投骰、不行动的规则。

## 非目标

- 不自动串联第一、第二半场。
- 不改变 JSON `starting_state` 格式或校验要求。
- 不改变行动顺序骰和移动骰是两次独立投骰的规则。
- 不重写技能系统或引入新的注册框架。

## 方案

采用保守扩展现有 `RaceEngine` 的方案。

### 默认起点堆叠

`RaceEngine` 初始化时仍可用空状态表示尚未应用默认开局。进入第 1 回合时，如果 `RaceConfig.starting_state is None`，且普通团子尚未被默认堆叠，则：

1. 使用现有 `build_round_order(1)` 逻辑产生第一回合行动顺序。
2. 将该顺序反转后作为 0 号地块 stack 存入 `RaceState`，保证最先行动者位于最上面。
3. 标记默认起点堆叠已完成，避免重复应用。
4. 后续本回合继续使用同一份行动顺序，不再二次投行动顺序骰。

这样第一回合的行动顺序、起点堆叠和实际行动循环使用同一组顺序骰结果。

### 回合开始时序

第一回合无 `starting_state` 时的回合 setup 顺序调整为：

1. 设置 `state.round_number = 1`。
2. 若需要默认开局，则投第一回合行动顺序骰，并按该行动顺序创建 0 号地块堆叠。
3. 调用 `start_round(1)`，触发所有 `on_round_start` 技能。
4. 过滤本回合被跳过的团子。
5. 使用步骤 2 已生成的行动顺序，并移除被跳过者。
6. 预投本回合移动骰。
7. 按顺序行动。

第二回合及以后、以及传入 `starting_state` 的比赛，继续使用现有回合 setup 思路：回合开始技能先执行，然后生成本回合行动顺序和移动骰。

### 弗洛洛技能

`PhrolovaSkill` 改为在 `on_round_start` 阶段判定：

- 如果弗洛洛未入场，不触发。
- 如果弗洛洛所在堆叠少于两个团子，不触发。
- 如果弗洛洛不是该堆叠底部，不触发。
- 满足条件时，记录弗洛洛本回合拥有一次移动加成。

行动时仍通过 `before_move` 消费记录并给 `context.movement += 3`。`before_move` 不再重新检查堆叠状态，只检查本回合是否已有回合开始判定产生的加成记录。消费后清除记录，防止同一回合重复生效。

该设计保留现有 `before_move` 作为“修改当前移动值”的落点，同时把规则判定点移动到回合开始，减少对引擎移动流程的侵入。

## 数据与状态

`RaceEngine` 需要新增少量状态：

- 默认起点堆叠是否已应用。
- 第 1 回合默认起点堆叠生成的行动顺序缓存，仅用于同一回合实际行动，避免重复投顺序骰。

`PhrolovaSkill` 可新增内部状态：

- 已获得回合开始加成的回合号集合或单个 `pending_round`。

由于模拟器通过 `config_factory` 为每次模拟创建新配置，并且引擎会 deepcopy participants，技能状态仍保持每场比赛隔离。

## 测试

需要新增或调整以下测试：

- 无 `starting_state` 时，第一回合按行动顺序反向堆叠到 0 号地块，最先行动者在最上面。
- 第一回合默认起点堆叠发生在 `on_round_start` 之前，回合开始技能能观察到完整起点堆叠。
- 弗洛洛在回合开始位于堆叠底部且堆叠至少两人时，本回合行动获得 +3。
- 弗洛洛如果回合开始时未在底部、单独在堆叠中、或未入场，则不获得 +3。
- 弗洛洛加成按回合开始状态决定；如果回合开始后堆叠变化，已获得的本回合加成仍在弗洛洛行动时生效。
- 传入 `starting_state` 时不应用默认起点堆叠。
- 布大王仍不参与前两回合行动顺序骰和移动骰。
